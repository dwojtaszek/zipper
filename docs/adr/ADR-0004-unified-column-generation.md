# ADR-0004: Unified Column Value Generation

## Status: Accepted

## Context

Prior to #262, column values were generated by two separate systems:
- `MetadataRowBuilder` for the `--with-metadata` path
- `DataGenerator` for the `--column-profile` path

This duplication meant adding a new column type required changes in two places.

## Decision

All column values are generated by `IColumnValueGenerator` implementations registered in `ColumnValueGeneratorRegistry`. `MetadataRowBuilder` is retired. `DataGenerator` is the sole entry point for row generation.

## Consequences

- Adding a new column type = one `IColumnValueGenerator` class + one registry entry
- Do not re-introduce per-mode value generators
- Legacy metadata kinds (`foldercustodian`, `filedatasize`, etc.) are implemented as `IColumnValueGenerator` instances, not special-case code
